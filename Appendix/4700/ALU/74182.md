The `74x182` is the carry look-ahead generator for 4 adders in the 74xx logic series.

This note documents a **functionally equivalent CMOS PSpice model** for the package-level device rather than the original TTL transistor network.

## What Is Included

- a top-level `.SUBCKT` named `CMOS_74182`
- behavioral carry-look-ahead equations for 4 propagate/generate stages
- one packaged device with intermediate carries and group propagate/generate outputs

## Model Scope

- The model is a functionally equivalent CMOS implementation of the `74x182` function, not an electrical model of a specific TTL or CMOS vendor family.
- Inputs `P0..P3` and `G0..G3` are assumed to be active-high propagate and generate signals from 4 lower-level adder slices.
- Outputs `C1`, `C2`, and `C3` are the anticipated carries into stages 1, 2, and 3 respectively, relative to incoming carry `C0`.
- `PG` and `GG` are the 4-slice group propagate and group generate signals.
- `C4` is the anticipated carry out of the 4-slice group.
- The port order is logical and grouped by carry input, bit-slice P/G inputs, and outputs.

## Port Order

```cir
.SUBCKT CMOS_74182
+ C0
+ P0 G0 P1 G1 P2 G2 P3 G3
+ C1 C2 C3 PG GG C4
+ VDD VSS
```

## Function

The carry-look-ahead equations are:

- `C1 = G0 or (P0 and C0)`
- `C2 = G1 or (P1 and G0) or (P1 and P0 and C0)`
- `C3 = G2 or (P2 and G1) or (P2 and P1 and G0) or (P2 and P1 and P0 and C0)`
- `GG = G3 or (P3 and G2) or (P3 and P2 and G1) or (P3 and P2 and P1 and G0)`
- `PG = P3 and P2 and P1 and P0`
- `C4 = GG or (PG and C0)`

## Netlist

> [!note]
> This model is **functionally equivalent**, not transistor-accurate to the original TTL family member.

**74182.cir**
```cir
* Functional CMOS 74x182 model.

.SUBCKT CMOS_74182
+ C0
+ P0 G0 P1 G1 P2 G2 P3 G3
+ C1 C2 C3 PG GG C4
+ VDD VSS

EPG PG VSS VALUE = {
+ IF((V(P0,VSS)>2.5)&(V(P1,VSS)>2.5)&(V(P2,VSS)>2.5)&(V(P3,VSS)>2.5),
+   V(VDD,VSS),
+   0)
+ }

EGG GG VSS VALUE = {
+ IF(
+   (V(G3,VSS)>2.5)|
+   ((V(P3,VSS)>2.5)&(V(G2,VSS)>2.5))|
+   ((V(P3,VSS)>2.5)&(V(P2,VSS)>2.5)&(V(G1,VSS)>2.5))|
+   ((V(P3,VSS)>2.5)&(V(P2,VSS)>2.5)&(V(P1,VSS)>2.5)&(V(G0,VSS)>2.5)),
+   V(VDD,VSS),
+   0)
+ }

EC1 C1 VSS VALUE = {
+ IF((V(G0,VSS)>2.5)|((V(P0,VSS)>2.5)&(V(C0,VSS)>2.5)), V(VDD,VSS), 0)
+ }

EC2 C2 VSS VALUE = {
+ IF(
+   (V(G1,VSS)>2.5)|
+   ((V(P1,VSS)>2.5)&(V(G0,VSS)>2.5))|
+   ((V(P1,VSS)>2.5)&(V(P0,VSS)>2.5)&(V(C0,VSS)>2.5)),
+   V(VDD,VSS),
+   0)
+ }

EC3 C3 VSS VALUE = {
+ IF(
+   (V(G2,VSS)>2.5)|
+   ((V(P2,VSS)>2.5)&(V(G1,VSS)>2.5))|
+   ((V(P2,VSS)>2.5)&(V(P1,VSS)>2.5)&(V(G0,VSS)>2.5))|
+   ((V(P2,VSS)>2.5)&(V(P1,VSS)>2.5)&(V(P0,VSS)>2.5)&(V(C0,VSS)>2.5)),
+   V(VDD,VSS),
+   0)
+ }

EC4 C4 VSS VALUE = {
+ IF((V(GG,VSS)>2.5)|((V(PG,VSS)>2.5)&(V(C0,VSS)>2.5)), V(VDD,VSS), 0)
+ }
.ENDS
```

## Example Usage

```cir
.include "74182.cir"

VDD vdd 0 DC 5
VC0 c0 0 DC 0
VP0 p0 0 DC 5
VG0 g0 0 DC 0
VP1 p1 0 DC 5
VG1 g1 0 DC 5
VP2 p2 0 DC 0
VG2 g2 0 DC 0
VP3 p3 0 DC 5
VG3 g3 0 DC 0

XU1
+ c0
+ p0 g0 p1 g1 p2 g2 p3 g3
+ c1 c2 c3 pg gg c4
+ vdd 0
+ CMOS_74182

.op
.probe V(c1) V(c2) V(c3) V(pg) V(gg) V(c4)
.end
```
