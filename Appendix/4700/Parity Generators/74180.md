The `74x180` is the 8-bit odd/even parity generator/checker in the 74xx logic series.

This note documents a **functionally equivalent CMOS PSpice model** for the package-level device rather than the original TTL transistor network.

## What Is Included

- a top-level `.SUBCKT` named `CMOS_74180`
- behavioral parity-reduction logic for 8 data inputs
- one packaged device with odd/even cascade-state inputs and outputs

## Model Scope

- The model is a functionally equivalent CMOS implementation of the `74x180` function, not an electrical model of a specific TTL or CMOS vendor family.
- The original device is best viewed as an 8-bit parity stage with a 2-wire odd/even state interface for cascading.
- The two control inputs are modeled as an encoded incoming parity state:
  - `EIN = 1`, `OIN = 0` means even parity so far
  - `EIN = 0`, `OIN = 1` means odd parity so far
- Standalone operation uses `EIN = 1` and `OIN = 0`.
- The port order is logical and grouped by data, control, and result signals.

## Port Order

```cir
.SUBCKT CMOS_74180
+ A B C D E F G H
+ EIN OIN EOUT OOUT
+ VDD VSS
```

## Function

Let `P = A xor B xor C xor D xor E xor F xor G xor H`.

Then:

- if `P = 0`, the outgoing parity state is unchanged
- if `P = 1`, the outgoing odd/even state is swapped

Equivalently:

- `EOUT = (EIN and not P) or (OIN and P)`
- `OOUT = (OIN and not P) or (EIN and P)`

For standalone 8-bit generation with `EIN = 1` and `OIN = 0`:

- `OOUT` is the even-parity bit
- `EOUT` is the odd-parity bit

## Netlist

> [!note]
> This model is **functionally equivalent**, not transistor-accurate to the original TTL family member.

**74180.cir**
```cir
* Functional CMOS 74x180 model.

.SUBCKT CMOS_74180
+ A B C D E F G H
+ EIN OIN EOUT OOUT
+ VDD VSS

EPAR PAR VSS VALUE = {
+ MOD(
+   IF(V(A,VSS)>2.5,1,0) + IF(V(B,VSS)>2.5,1,0) +
+   IF(V(C,VSS)>2.5,1,0) + IF(V(D,VSS)>2.5,1,0) +
+   IF(V(E,VSS)>2.5,1,0) + IF(V(F,VSS)>2.5,1,0) +
+   IF(V(G,VSS)>2.5,1,0) + IF(V(H,VSS)>2.5,1,0),
+   2)
+ }

EEOUT EOUT VSS VALUE = {
+ IF(V(PAR,VSS)<0.5,
+   V(EIN,VSS),
+   V(OIN,VSS))
+ }

EOOUT OOUT VSS VALUE = {
+ IF(V(PAR,VSS)<0.5,
+   V(OIN,VSS),
+   V(EIN,VSS))
+ }
.ENDS
```

## Example Usage

```cir
.include "74180.cir"

VDD vdd 0 DC 5
VA a 0 DC 1
VB b 0 DC 0
VC c 0 DC 1
VD d 0 DC 1
VE e 0 DC 0
VF f 0 DC 0
VG g 0 DC 1
VH h 0 DC 0
VEIN ein 0 DC 5
VOIN oin 0 DC 0

XU1
+ a b c d e f g h
+ ein oin eout oout
+ vdd 0
+ CMOS_74180

.op
.probe V(eout) V(oout)
.end
```
