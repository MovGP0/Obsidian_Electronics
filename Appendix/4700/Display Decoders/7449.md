The `74x49` is the BCD-to-seven-segment decoder/driver for LED displays in the 74xx logic series.

This note documents a **functionally equivalent CMOS PSpice model** for the package-level device rather than the original TTL transistor network.

## What Is Included

- a top-level `.SUBCKT` named `CMOS_7449`
- a behavioral BCD-to-seven-segment decoder using open-collector outputs
- one packaged device with a simplified active-low blanking input

## Model Scope

- The model is a functionally equivalent CMOS implementation of the `74x49` function, not an electrical model of a specific TTL or CMOS vendor family.
- Legacy `74x49` references vary between the earlier 7449 and later LS49-family package details. This note models the shared decoder behavior with one active-low blanking input and open-collector segment outputs.
- Outputs are active-low open-collector segment drivers intended for common-anode LED displays and therefore require external pull-up or LED load paths.
- This simplified model decodes valid BCD digits `0` through `9`. Inputs `10` through `15` are blanked instead of reproducing the legacy non-decimal glyphs.
- The historical non-tailed `6` and `9` font associated with the `74x49` family is preserved in the segment decode.

## Port Order

```cir
.SUBCKT CMOS_7449
+ D C B A BIN
+ AN BN CN DN EN FN GN
+ VDD VSS
```

## Function

- if `BI_N = 0`, all segment outputs are off
- else the BCD input is decoded to active-low segment outputs for digits `0` through `9`

> This model is **functionally equivalent**, not transistor-accurate to the original TTL family member.

## Example Usage

```cir
.include "7449.cir"

VDD vdd 0 DC 5
RSEG_A an vdd 1K
RSEG_B bn vdd 1K
RSEG_C cn vdd 1K
RSEG_D dn vdd 1K
RSEG_E en vdd 1K
RSEG_F fn vdd 1K
RSEG_G gn vdd 1K

VD d 0 DC 0
VC c 0 DC 1
VB b 0 DC 0
VA a 0 DC 1
VBI bin 0 DC 5

XU1
+ d c b a bin
+ an bn cn dn en fn gn
+ vdd 0
+ CMOS_7449

.op
.probe V(an) V(bn) V(cn) V(dn) V(en) V(fn) V(gn)
.end
```
