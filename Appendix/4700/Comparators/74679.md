The `74x679` is the 12-bit address comparator in the 74xx logic series.

This note documents a **functionally equivalent CMOS PSpice model** for the package-level device rather than the original TTL transistor network.

## What Is Included

- a top-level `.SUBCKT` named `CMOS_74679`
- behavioral logic for the programmed 12-bit boundary-compare function
- one packaged device with active-low enable and active-low match output

## Model Scope

- The model is a functionally equivalent CMOS implementation of the `74x679` function, not an electrical model of a specific TTL or CMOS vendor family.
- The TTL device compares a 12-bit address word against a boundary pattern selected by the 4-bit `P` input field. For code `n`, bits `A1` through `An` must be LOW and bits above that boundary must be HIGH.
- Only programming codes `0` through `12` are meaningful in this simplified model; codes `13` through `15` never assert a match.
- The output is active low and only responds when the active-low enable input is asserted.
- The port order is logical and grouped by control, address, and program inputs.

## Port Order

```cir
.SUBCKT CMOS_74679
+ GN
+ A1 A2 A3 A4 A5 A6 A7 A8 A9 A10 A11 A12
+ P0 P1 P2 P3
+ YN
+ VDD VSS
```

## Function

- decode the positive-logic `P[3:0]` field as a boundary value `n`
- if `G_N = 0` and the address pattern is `A1..An = 0` and `A(n+1)..A12 = 1`, then `Y_N = 0`
- otherwise `Y_N = 1`

## Example Usage

```cir
.include "74679.cir"

VDD vdd 0 DC 5
VG gn 0 DC 0
VP0 p0 0 DC 1
VP1 p1 0 DC 1
VP2 p2 0 DC 1
VP3 p3 0 DC 0

XU1
+ gn
+ 0 0 0 0 0 0 0 vdd vdd vdd vdd vdd
+ p0 p1 p2 p3
+ yn
+ vdd 0
+ CMOS_74679

.op
.probe V(yn)
.end
```
